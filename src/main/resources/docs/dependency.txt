다양한 의존관계 주입 방법
- 생성자 주입
- 수정자 주입 (setter 주입)
- 필드 주입
- 일반 메서드 주입

생성자 주입
- 이름 그대로 생성자를 통해서 의존 관계를 주입 받는 방법이다.
- 지금까지 우리가 진행했던 방법이 생성자 주입
- 툭징
    - 생성자 호출 시점에 딱 1번만 호출되는 것이 보장
    - 불변, 필수 의존관계에 사용
*중요*
- 생성자가 딱 1개만 있으면 @Autowired 를 생략해도 자동 주입 (스프링 빈에 한정)

수정자 주입 (setter)
- setter 라 불리는 필드의 값을 변경하는 수정자 메서드를 통해서 의존관계 주입
- 특징
    - 선택, 변경 가능성이 있는 의존관계에 사용
    - 자바빈 프로퍼티 유약의 수정자 메서드 방식을 사용하는 방법
